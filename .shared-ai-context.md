## OpenSpec（選用）

- 用於中大型 / 高風險變更的規格對齊（OpenSpec / OPSX）

- 可跳過：小修（文案/格式/註解）、低風險單點 bug fix、純文件、純測試/lint/format
- 需要時先 `/opsx:new`；實作中用 `/opsx:ff` 推進；完成後 `/opsx:apply` / `/opsx:archive`
- 節省 token：僅在符合觸發條件或使用者明確提到 OpenSpec/opsx 時再讀取 `openspec` skill

## 更新規則

**每位 AI 助手完成工作後，必須更新此檔案：**

### Git 歷史（機械同步）
- 執行 `git log --oneline --graph -15` 並**原樣貼上**到「Git 歷史」區塊
- 純粹反映 repo 狀態，不需人工編輯

### 變更日誌（語義紀錄）
變更日誌的定位是**比 git history 更完整的審計軌跡**，涵蓋所有 AI 操作——不論是否產生 commit。

**必須記錄的情境：**
- 新增/修改/刪除檔案（含 commit hash 如有）
- 設定變更（.env、config 等可能被 gitignore 的檔案）
- 本地操作（git sync、branch merge、環境設定）
- 重要決策或討論結論（例：選擇某技術方案的原因）

**格式：** `[日期] [AI名稱]: 變更摘要`

### 其他區塊
- 新增/修改檔案 → 更新「專案資訊 / 主要內容」等相關區塊
